2. Was Redline nicht tut

  • Keine Cloud-Telemetrie, kein Google Analytics, kein Verkauf personenbezogener Daten
  • Kein Upload von Spielständen, Passwörtern, Dokumenten oder Chat-Inhalten
  • Kein stilles Fernsteuern deines PCs

3. Lokale Daten (auf deinem PC)

Redline speichert Einstellungen und Protokolle unter:

%AppData%\RedlineGamingOptimizer\

  • settings.json — Sprache, Theme, Scan-Tiefe, Lizenzstatus (lokal)
  • changes.log — Protokoll durchgeführter Optimierungen (für Übersicht/Rücksetzen)
  • Backups z. B. service-tweaks-backup.json, Registry-Backup für deaktivierte Autostarts

4. „Telemetry“ in der App

Der Begriff bezeichnet nur lokale Hardware-Auslesung (CPU/GPU/RAM-Temperatur und Auslastung) über LibreHardwareMonitor/WMI — ausschließlich für Dashboard-Anzeige. Keine Übertragung an Redline-Server.

5. Netzwerk (nur bei deiner Aktion)

  • Update-Check: HTTPS zu GitHub (version.json, Releases) — nur wenn du Update prüfst oder Auto-Update aktiv ist
  • Treiber-Links: Browser öffnet Hersteller-Seiten (NVIDIA, AMD, Intel, …)
  • DNS-Benchmark: Ping zu öffentlichen DNS-Servern zur Messung — keine dauerhafte Umleitung ohne Bestätigung

6. Rechte (DSGVO)

Da keine zentrale Nutzerdatenbank existiert, liegen personenbezogene Daten praktisch nur auf deinem Gerät. Du kannst den Ordner löschen oder die App deinstallieren. Anfragen: Support.

English summary

Redline works locally. Hardware “telemetry” is on-device sensor reading only. Update checks use GitHub when you request them. No cloud profiling. Data folder: %AppData%\RedlineGamingOptimizer\.
